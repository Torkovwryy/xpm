# Security Policy

## Supported Versions

Currently, the XPM project is under active initial development. Security updates are applied only to the `main` branch and future stable release channels.

| Version | Supported          |
| ------- | ------------------ |
| < 1.0.0 | :white_check_mark: |

## Reporting a Vulnerability

We take the security of the XPM package manager very seriously, as it sits at the core of the software supply chain.

If you discover a vulnerability, **please do NOT open a public issue.**

Instead, report it using GitHub's Private Vulnerability Reporting.

### Triage Process

1. A maintainer will acknowledge receipt of the vulnerability within 48 hours.
2. The team will investigate and determine if the vulnerability is valid.
3. If valid, a patch will be developed in a private fork.
4. A CVE may be requested based on the severity.
5. The patch will be released, and the reporter will be credited (if desired).
